Executive Summary  
The Visiting Card Scanner application will allow users to capture business card details either through a device camera or by  
uploading an image/PDF. The system will extract information such as person name, designation, phone number, email,  
company name, website, address, and other available fields. It will then optionally enrich the extracted company information  
using web search/company lookup logic and assign an appropriate business category such as Electrical Wires Supplier,  
Corporate Gifts Vendor, Packaging Supplier, IT Services, Real Estate, or other defined categories. The initial product will be a  
public web tool with no login, but the architecture should be designed so it can evolve into a SaaS product and later support an  
Android app using the same backend APIs.

7. Company Enrichment and Category Assignment  
The system should not rely only on the text printed on the card. If company name, website, or email domain is available, the  
backend should attempt to infer the company activity and suggest a category. In Phase 1, use a hybrid approach: rules first,  
then AI/web enrichment if enabled.  
Input Signal  
Company name  
How to Use It  
Search or match against known keywords. Example: "ABC Cables" may  
suggest wires/cables/electrical category.  
Fetch homepage meta title/description where allowed, or use search  
snippets.  
If email is name@company.com, derive company.com and use it for  
enrichment.  
Website/domain  
Email domain  
Address/city  
Card keywords  
Useful for regional reports and duplicate detection.  
Use product/service words printed on card: cables, wires, packaging,  
interiors, logistics, software, printing, etc.  
7.1 Category Assignment Logic  
CATEGORY ENGINE LOGIC  
1. Extract keywords from OCR text, company name, website text, and enrichment result.  
2. Normalize words: lowercase, remove punctuation, use synonyms.  
3. Match against category keyword dictionary.  
4. If multiple categories match, rank by:  
- keyword match count  
- keyword importance  
- source reliability: website > card text > search snippet > guessed domain  
- AI confidence score, if AI model is used  
5. Assign category only if confidence is above configured threshold.  
6. If confidence is low, mark as "Uncategorized" and ask user/admin to select category manually.  
7. Store category reason for transparency.  
Category Output  
suggested_category  
confidence  
Description  
The category name selected by the system.  
Numeric score from 0 to 1.  
reason  
needs_review  
Human-readable reason, e.g., "Matched keywords: cable, wire, electrical".  
True when confidence is low or multiple categories conflict.
